# MoE-YOLO 方案设计（级联 + 轻门控专家）

## 1. 目标与约束

### 1.1 业务目标
- 在普通 Android 设备上实时运行。
- 支持道路障碍、路标、盲道等多类目标识别。
- 支持后续扩展到室内导航。

### 1.2 工程约束
- 当前数据存在明显类别不均衡。
- 小目标占比高，直接单模型训练容易造成远距目标漏检。
- 移动端部署受限于延迟、功耗、内存和算子支持。

## 2. 推荐方案概述

推荐采用两阶段演进路径：

1. 第一阶段：级联检测（Cascade）
- Always-on 主模型：轻量化 YOLO，全类别覆盖，保证实时召回。
- 触发式专家模型：在疑难样本触发时执行，用于补精度。

2. 第二阶段：轻量 MoE 化
- 在共享 backbone 的前提下引入轻量 router。
- 只稀疏激活专家 head，避免全模型动态路由的部署复杂度。

## 3. 架构设计

### 3.1 在线推理流程

1. 主模型推理（Stage-1）
- 输入整帧图像。
- 输出初始检测框集合 D1。

2. 触发判定（Trigger Policy）
- 根据 D1 的统计特征判断是否触发专家模型：
  - 低置信目标比例是否过高。
  - 小目标（面积比）数量是否过多。
  - 关键类别是否缺失或不稳定。

3. 专家选择（Routing）
- Rule-based 或 Tiny MLP Router 从候选专家中选 top-k。
- 推荐从 top-1 起步，再扩展到 top-2。

4. 专家推理（Stage-2）
- 仅执行被选中的专家模型。
- 输出专家检测框集合 D2。

5. 融合输出（Merge）
- 执行按类别 NMS 或 WBF。
- 得到最终检测结果 D。

### 3.2 专家划分建议

- Expert-Ground：地面大区域类（盲道、横道）。
- Expert-TinyObstacle：小尺寸障碍类（cone、barrier 等）。
- Expert-Indoor（预留）：室内导航相关目标。

### 3.3 触发策略建议（可直接上线）

触发条件建议使用 OR 逻辑：
- 条件 A：低置信度目标比例 > r_low。
- 条件 B：小目标数量 > n_tiny。
- 条件 C：主模型输出框数量异常（过少或过多）。

推荐初始阈值：
- r_low = 0.5（置信度阈值 0.35）
- n_tiny = 4（面积比阈值 0.01）

## 4. 训练设计

### 4.1 Stage-1 主模型训练
- 使用现有 merged 数据训练统一主模型。
- 重点配置：
  - 类别重采样（class-aware / repeat factor）。
  - 小目标增强（copy-paste、small object oversampling）。
  - 多尺度训练（包含更高分辨率阶段）。

### 4.2 Stage-2 专家模型训练
- 按类别簇构建子数据集。
- 每个专家在各自重点类别上强化训练。
- 主模型和专家模型可共享初始化权重，减少训练成本。

### 4.3 Router 训练（可选）
- 输入特征：主模型输出统计（框数量、平均置信度、小目标比率、类别直方图）。
- 标签：该帧是否触发专家 / 触发哪个专家可提升指标。
- 目标函数：二分类（触发与否）或多标签分类（专家选择）。

## 5. 评估体系

### 5.1 精度指标
- mAP50 与 mAP50-95。
- 每类 AP（重点看长尾类与小目标类）。
- 小目标 AP_S。

### 5.2 部署指标
- 端侧平均延迟与 P95 延迟。
- FPS 与峰值内存。
- 功耗与温升（长时运行）。

### 5.3 触发收益指标
- 触发率（专家被触发比例）。
- 触发后增益（mAP 改善 / 延迟增量）。
- 性价比：Delta(mAP) / Delta(latency)。

## 6. 可行性判断

1. 技术可行：
- 级联路径不依赖复杂动态算子，部署风险低。

2. 工程可控：
- 可先做规则触发，再平滑替换为学习型 router。

3. 迭代友好：
- 可按阶段 A/B 验证，避免一次性大改导致回归难定位。

## 7. 风险与对策

1. 风险：触发过于频繁导致实时性下降。
- 对策：限制 top-k、设置触发冷却窗口、对触发阈值做温和上调。

2. 风险：专家模型与主模型输出冲突。
- 对策：统一后处理策略（按类别 NMS），保留高置信结果。

3. 风险：专家收益不足。
- 对策：基于误检漏检样本重建专家数据集，按类别簇迭代。

## 8. 里程碑

- M1：主模型 baseline + 触发规则上线。
- M2：接入两个专家模型并完成融合评估。
- M3：训练 Router，替换规则触发。
- M4：Android 端部署压测与稳定性优化。

## 9. 本目录代码说明

- `moe_yolo/config.py`：配置数据结构。
- `moe_yolo/router.py`：规则触发与轻量 router。
- `moe_yolo/cascade.py`：级联推理管线（主模型 + 专家 + 融合）。
- `moe_yolo/train_router_stub.py`：router 训练脚手架。

该实现是可执行的初步版本，用于验证触发策略和收益曲线，不是最终生产实现。
